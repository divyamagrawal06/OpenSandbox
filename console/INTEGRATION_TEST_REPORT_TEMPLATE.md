# Developer Console Integration Verification (Phase 1)

Use this checklist and artifact set in your PR to prove the console + server auth/authz integration works end-to-end.

## Automated Browser Coverage

Run from `console/`:

```bash
npx playwright install chromium
npm run test:e2e
```

The suite covers:

1. Sandbox list renders with API data.
2. Detail flow works (endpoint retrieval, renew request).
3. Create flow works and redirects to created sandbox detail.
4. Auth misconfiguration (`MISSING_TRUSTED_IDENTITY`) renders explicit banner.

## Required Screenshots (generated by Playwright)

The E2E suite writes screenshots to:

- `console/output/playwright/console-list.png`
- `console/output/playwright/console-detail.png`
- `console/output/playwright/console-create.png`
- `console/output/playwright/console-auth-error.png`

Attach these images in your PR description as proof.

## Manual Validation (server + console)

### 1) Start server (Phase 1 dual auth)

Use config values consistent with OSEP-0006:

- `auth.mode = "api_key_and_user"`
- `auth.user_mode = "trusted_header"`
- `authz.default_role = "read_only"` (or as desired)

### 2) Start console

```bash
cd console
VITE_DEV_IDENTITY_USER=alice VITE_DEV_IDENTITY_ROLES=operator npm run dev
```

### 3) Verify operator behavior

- Can list/get/endpoint.
- Can create/renew/delete/pause/resume.
- Mutations appear in server logs with `mutation_audit`.

### 4) Verify read_only behavior

Run with:

```bash
VITE_DEV_IDENTITY_USER=alice VITE_DEV_IDENTITY_ROLES=read_only npm run dev
```

- UI hides mutation actions.
- Direct mutation API calls return `403` (`INSUFFICIENT_ROLE`).

### 5) Verify auth misconfiguration behavior

- Remove proxy-injected trusted headers.
- Confirm requests fail with `401` (`MISSING_TRUSTED_IDENTITY`).
- Confirm console shows explicit auth-required banner.

## Evidence to include in PR

- E2E command output (`npm run test:e2e`).
- All screenshots listed above.
- Server test summary (e.g. `uv run pytest tests/ -q`).
- Brief note confirming no API key is exposed in browser client code.
